# Cppcheck suppressions

Cppcheck 2.7 rejects comments in `--suppressions-list` files, so
`.cppcheck-suppressions` is intentionally machine-readable only. Document each
entry here with a reason and owner.

| Suppression | Reason | Owner |
| --- | --- | --- |
| `comparePointers:radio/src/thirdparty/CMSIS/Include/cmsis_gcc.h` | CMSIS GCC intrinsic helpers intentionally compare linker/register helper addresses in a third-party vendor header. | firmware-safety |
| `preprocessorErrorDirective:radio/src/datastructs_private.h` | Cppcheck cannot expand the project `PACK` macro when preprocessor directives appear in the packed declaration body. | firmware-safety |
| `preprocessorErrorDirective:radio/src/pulses/pulses.h` | Cppcheck cannot expand the project `PACK` macro when preprocessor directives appear in the packed declaration body. | firmware-safety |
| `preprocessorErrorDirective:radio/src/targets/common/arm/stm32/rotary_encoder_driver.cpp` | Cppcheck does not see the STM32 LL SYSCFG clock macro selected by the successful target build, so it reaches the compile-time guard intended for unsupported MCUs. | firmware-safety |
| `preprocessorErrorDirective:radio/src/targets/common/arm/stm32/stm32_gpio.cpp` | Cppcheck does not see the STM32 LL SYSCFG clock macro selected by the successful target build, so it reaches the compile-time guard intended for unsupported MCUs. | firmware-safety |
| `preprocessorErrorDirective:radio/src/targets/common/arm/stm32/usb_driver.cpp` | Cppcheck does not see the STM32 LL SYSCFG clock macro selected by the successful target build, so it reaches the compile-time guard intended for unsupported MCUs. | firmware-safety |
| `uninitvar:radio/src/thirdparty/stb/stb_image.h` | STB image decoder is third-party code; firmware gates track first-party defects separately from vendored decoder internals. | firmware-safety |
| `nullPointerArithmeticOutOfMemory:radio/src/gui/colorlcd/libui/bitmapbuffer.cpp` | Existing color LCD bitmap allocation path predates Edge16 safety tooling; tracked as baseline so new cppcheck errors remain blocking. | firmware-safety |
